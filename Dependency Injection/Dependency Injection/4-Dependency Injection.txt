What is the problem??
--> we have to recieve the object of the service class that the IOC Container supplied

Solution--> implement Dependency Injection (DI) 

(*)DI-->Design Pattern that Implements IOC (Inversion OF Control)

In Brief--> IOC Supplies & DI Recieves

--> controller wil recieve the service object either in -->1- Constructor Parameter
														  2- Method Parameter
														  
1-Constrcutor injection--> if you want to use the object in more than 1 method in your controller



2-method injetion--> if you will use the service object in a only specific method
##NOTES-->justadd a [FromServices] behind the ICityService object in your method parameter

@inject Interface_name obj_name
